# Sports Team Stats 🏆

**Технологии**  
- Python 3.9+
- SQLite3
- PyTest
- Docker (опционально, не запущен из-за аппаратных ограничений ноутбук)

## Установка и запуск

### Обычный запуск
```bash
pip install -r requirements.txt
python run.py
```

> В Visual Studio можно запускать через `python run.py`, программа создаст базу `sports.db` при первом запуске.

### Тесты
```bash
python -m pytest tests -v
# или конкретный тест
python -m pytest tests/test_match.py -v
```

### Docker (опционально)
Docker-файл подготовлен, однако запуск контейнера не выполнялся по техническим причинам — старый процессор ноутбука не поддерживает виртуализацию, необходимую для Docker.
В остальном структура проекта полностью готова к контейнеризации.

При возможности запуск выглядел бы так:

```bash
docker build -t sports-team .
docker run -it --rm sports-team
```

## Основные классы
- `Player` — абстрактный класс игрока (имя, номер, статистика).  
- `Forward`, `Defender`, `Goalkeeper` — наследники `Player` с собственной логикой.  
- `Team` — команда (набор игроков, вычисление командной статистики).  
- `Match` — матч (учёт забитых/пропущенных голов, распределение очков).  
- `DB` / `db.py` — модуль работы с SQLite (создание таблиц, сохранение и чтение статистики).  
- `report.py` — генерация отчётов в `.docx` (python-docx).

## Особенности
- Модульная структура: пакет `sports_team` с несколькими модулями.  
- ООП: абстрактный базовый класс, наследование, полиморфизм.  
- Инкапсуляция через `@property` и `setter`.  
- Dunder-методы (`__init__`, `__str__`, `__repr__`, `__eq__`).  
- Декораторы для утилит (логирование/тайминг).  
- Интеграция с SQLite для персистентности.  
- Покрытие тестами с использованием `pytest`.  
- Генерация отчёта `.docx`

## Использование (пример взаимодействия через меню)

После запуска отображается главное меню:

```
===== МЕНЮ =====
1. Создать команду
2. Добавить игрока в команду
3. Показать информацию о команде
4. Провести матч
5. Сохранить отчёт (.docx)
6. Сохранить всё в базу данных
7. Показать все команды
8. Очистить все данные о командах
9. Открыть базу данных
0. Выход
```

### Пример работы:

-- После выбора 1 создаётся новая команда, куда можно добавлять игроков (пункт 2).

-- При выборе 4 программа моделирует матч между командами.

-- Пункт 5 сохраняет отчёт о текущих результатах в .docx.

-- Пункт 6 записывает данные в SQLite.
